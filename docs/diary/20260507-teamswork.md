# セッション記録: 2026-05-07（Teamswork / 自律協働メモ）

## 目的

サブエージェントとメインエージェントが巨大な教本増設作業を最後まで自律完遂するための、運用上の注意点を整理する。

## うまくいった点

- OpenCode/oh-my-openagent の設定を見直し、エージェント構成を戻せた。
- サブエージェント用のコンテキスト運用（`maxTokens`、役割分担、分割統治）を文書化できた。
- 失敗や設定崩れが起きた際に、`docs/candi-ref/tech-stack.md` を参照元にできるようになった。

## 失敗した点と対処

### 1. エージェント設定の崩れ

- 症状: Sisyphus / Hephaestus / Atlas / Prometheus が一覧から消える。
- 対処: `.opencode/oh-my-openagent.jsonc` の JSONC 崩れを修正し、`plugin` 設定を復旧。

### 2. モデル切替時の設定不足

- 症状: 既知プロバイダがカスタム登録扱いになる。
- 対処: `enabled_providers` と provider 定義を再確認し、tech-stack に記録。

### 3. 自律作業の途中ハング / 失敗

- 症状: TODOシステムの503、設定の不整合、ビルドエラー。
- 対処: 直前の成功地点へ戻り、修正範囲を1点に絞って再実行。

## 今後の運用ルール

1. **コミットのタイミング**
   - 各フェーズ完了時にコミット。
   - テスト・ビルド通過後にプッシュ。
   - 失敗時は再現可能な最小単位で切る。

2. **Diaryの書き方**
   - 日の終わりだけでなく、**失敗・復帰・設定変更**の節目でも記録する。
   - ファイル名は `YYYYMMDD-teamswork.md` を基本にする。

3. **自動復帰の考え方**
   - ハングしたら、同じ依頼を丸投げで再送しない。
   - まず「最後に成功した状態」を記録する。
   - その後、責務を小さく分け直して再開する。

## 次回へのメモ

- 大きい作業は「メインが分割、サブが生成、メインが統合」の順に回す。
- サブエージェントは長文の一括生成ではなく、章単位・ファイル群単位で扱う。
- 失敗したら、まず記録してから直す。
