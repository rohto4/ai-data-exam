## 分類モデルの評価指標

### 定義と背景

機械学習モデルの評価は、モデルの性能を客観的に測定し、実務で活用できるかを判断する重要な工程である。分類タスクでは、単純な正解率だけでは不十分で、クラスの不均衡や誤分類のコストを考慮した複合的な評価が必要となる。

評価指標の選択は、ビジネス目的と密接に関連する。例えば、医療診断では見逃し（偽陰性）を最小化することが重要だが、スパム検出では誤検出（偽陽性）を最小化することが重要である。このような文脈に応じた指標選定が、実務でのモデル成功を左右する。

### 混同行列（Confusion Matrix）

混同行列は、分類モデルの予測結果を整理した表であり、すべての評価指標の基礎となる。

|  | 予測：陽性 | 予測：陰性 |
|--|-----------|-----------|
| **実際：陽性** | TP（真陽性） | FN（偽陰性） |
| **実際：陰性** | FP（偽陽性） | TN（真陰性） |

- **TP（True Positive）**：実際に陽性で、予測も陽性（正しく検出）
- **TN（True Negative）**：実際に陰性で、予測も陰性（正しく除外）
- **FP（False Positive）**：実際に陰性だが、予測が陽性（誤検出）
- **FN（False Negative）**：実際に陽性だが、予測が陰性（見逃し）

### 主要指標

**正解率（Accuracy）**：
- 計算：(TP + TN) / (TP + TN + FP + FN)
- 全予測のうち、正しく分類できた割合
- クラス不均衡時は信頼できない（陰性が90%のデータセットで、すべて陰性と予測しても90%正解率）
- 均衡データセットや、誤分類コストが対称的な場合に適している

**適合率（Precision）**：
- 計算：TP / (TP + FP)
- 予測した陽性のうち、実際に陽性だった割合
- FP（誤検出）を抑えたい時に重要
- スパム検出や、誤アラートのコストが高い場面で重視される
- 「予測が陽性のとき、どの程度信頼できるか」を示す

**再現率（Recall / Sensitivity）**：
- 計算：TP / (TP + FN)
- 実際の陽性のうち、予測できた割合
- FN（見逃し）を抑えたい時に重要
- 医療診断や、異常検知など、見逃しのコストが高い場面で重視される
- 「実際の陽性をどの程度捉えられたか」を示す

**F1スコア**：
- 計算：F1 = 2 × (Precision × Recall) / (Precision + Recall)
- PrecisionとRecallの調和平均
- バランスの取れた評価指標
- PrecisionとRecallのトレードオフを単一の値で表現
- クラス不均衡時にも安定した評価が可能

**特異度（Specificity）**：
- 計算：TN / (TN + FP)
- 実際の陰性のうち、正しく予測できた割合
- 陰性クラスのRecallに相当
- 医療検査の特異度や、正常例の検出精度を評価する際に使用

### ROC曲線とAUC

**ROC曲線（Receiver Operating Characteristic）**：
- 縦軸：真陽性率（Recall = TP / (TP + FN)）
- 横軸：偽陽性率（FPR = FP / (FP + TN) = 1 - Specificity）
- 分類閾値を変化させた時のトレードオフを可視化
- 左上に近いほど良い性能
- 閾値選定の指針となる

**AUC（Area Under the Curve）**：
- ROC曲線下の面積
- 0.5：ランダムと同じ（判別能力なし）
- 1.0：完璧な分類
- 0.8以上：良好、0.9以上：優秀
- 閾値に依存しない評価指標として広く使われる
- クラス不均衡時でも安定した評価が可能

**PR曲線（Precision-Recall Curve）**：
- 縦軸：Precision、横軸：Recall
- クラス不均衡時（陽性が少数）にROCより情報量が多い
- AUC-PR（Average Precision）で定量化

## 回帰モデルの評価指標

### 定義と背景

回帰モデルの評価は、予測値と実測値の差（誤差）をどのように定義・集約するかによって、異なる側面が強調される。ビジネス目的に応じて、外れ値の影響を受けやすい指標か、頑健な指標かを選定する必要がある。

**MAE（Mean Absolute Error）**：
- 絶対誤差の平均：MAE = Σ|yi - ŷi| / n
- 外れ値の影響を受けにくい（頑健）
- 単位が元のデータと同じで解釈しやすい
- 誤差の大きさに比例してペナルティを与える

**MSE（Mean Squared Error）**：
- 2乗誤差の平均：MSE = Σ(yi - ŷi)² / n
- 外れ値に敏感（大きな誤差に強いペナルティ）
- 単位が2乗になって解釈しにくい
- 数学的に扱いやすく、最適化で広く使われる

**RMSE（Root Mean Squared Error）**：
- MSEの平方根：RMSE = √MSE
- 単位が元のデータと同じで解釈しやすい
- 一般的に最も使われる回帰評価指標
- 外れ値に敏感だが、解釈性が良い

**R²（決定係数）**：
- モデルがデータの変動をどれだけ説明できるか
- R² = 1 - (Σ(yi - ŷi)² / Σ(yi - ȳ)²)
- 1に近いほど良い、0は単純な平均予測と同じ
- 0以下の場合は単純な平均予測より悪い
- 過学習すると訓練データで高い値が出るため、検証データでの評価が必須

**MAPE（Mean Absolute Percentage Error）**：
- 平均絶対パーセント誤差：MAPE = Σ|(yi - ŷi) / yi| / n × 100%
- パーセント表示で直感的
- 実測値が0に近いと不安定になる
- 需要予測などで広く使われる

## 交差検証（Cross-Validation）

### 定義と背景

交差検証は、限られたデータを効率的に活用し、モデルの汎化性能を信頼性高く評価する手法である。単純なホールドアウト検証では、データの分割方法によって評価結果が大きく変わる可能性があるが、交差検証は複数回の分割・評価を行うことで、このばらつきを軽減する。

**k-fold交差検証**：
- データをk個に分割（通常k=5または10）
- 1つをテスト、残りk-1つで学習
- k回繰り返して平均を取る
- データを効率的に活用でき、評価のばらつきを抑えられる
- 計算コストは単純分割のk倍

**層化k-fold（Stratified k-fold）**：
- クラス比率を維持して分割
- クラス不均衡時に推奨
- 各foldでクラス分布が全体と同じになるように分割
- 分類タスクでは標準的に使用される

**Leave-One-Out（LOO）**：
- 1つだけをテスト、残り全てで学習
- データ数分繰り返し
- 計算コストが高い（n回の学習が必要）
- 小規模データセットでのみ使用

**時系列交差検証**：
- 時系列データでは、未来のデータを使って過去を予測するような分割を避ける
- ウォークフォワード検証：過去データで学習、未来データで検証
- 時系列の構造を考慮した評価が必要

### 実務・試験論点

**評価指標の選択指針**：
- クラス均衡・誤分類コスト対称：Accuracy
- 誤検出コスト高（スパム検出など）：Precision
- 見逃しコスト高（医療診断など）：Recall
- バランス重視：F1スコア
- 閾値非依存・クラス不均衡：AUC-ROC、AUC-PR

**回帰指標の選択指針**：
- 外れ値に頑健：MAE
- 外れ値を重視：RMSE
- 解釈性重視：R²、MAPE

**交差検証の活用場面**：
- ハイパーパラメータ調整
- モデル選択
- 汎化誤差の推定
- データが少ない場合の安定した評価

**よくある誤解**：
- 「Accuracyが高い=良いモデル」（クラス不均衡時は誤り）
- 「PrecisionとRecallは同時に最大化できる」（トレードオフ関係）
- 「R²が高い=過学習していない」（訓練データでのR²は高くなりがち）
- 「交差検証は必須」（大規模データではホールドアウトでも十分な場合がある）

- Precision：誤検出を抑える、Recall：見逃しを抑える
- F1スコアはPrecisionとRecallのバランス
- AUCは閾値に依存しない評価
